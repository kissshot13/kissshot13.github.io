主机记录


即是域名解析的前缀
比如我的域名为 kissshot.xyz

值 含义
wwww 解析的域名为wwww.kissshot.xyz
@ 直接解析主域名kissshot.xyz
* 匹配*.kissshot.xyz
mail mail.kissshot.xyz通常用于游戏服务器

记录类型

值 含义
A 指定IP4地址，域名指向一个ip地址就需要A记录
AAAA 指定的IP6地址
CNAME 如果需要将域名指向另一个域名,再由另一个域名提供ip地址,就需要添加CNAME记录

why‘s hexo

hexo 是建站引擎，主要是快，毕竟是markdown

安装

  • 先安装git和node，前段没这两个工具怎么装逼呢
  • 安装hexonpm install -g hexo-cli

hexo建站

  • hexo init <folder>
  • cd <folder>

  • npm install
    大概就是这糗样：

    • scaffolds 模板，
    • source，资源，除了posts文件夹，``开头的都会被忽略
    • themes，主题，我使用的是next主题

配置

配置在config.yml里面，都比较简单，参考配置文档
我只贴出我关于git的配置，毕竟我搭载的blog基于github page

指令

请查阅官方文档hexo
列举几个常用的:

  • 新建文章 hexo new [layout] <title>
    • [layout]没有设置就使用默认的，常用post,page和draft.不想要layout可以设置Front-Matter 中的layout:设为 false
  • 草稿，有种特殊的布局draft,hexo publish [layout] <title>,这种布局在建立时会被保存到 source/_drafts 文件夹，您可通过 publish 命令将草稿移动到 source/_posts 文件夹
  • 生成静态文件，hexo g,
    • hexo generate --watch监视文件变化
  • 启动服务器 hexo s
  • 发布hexo d
    • hexo 3 需要先安装npm install hexo-server --save，-p 可以定义端口号
  • 上面两个命令可以简写$ hexo g -d
  • 清除hexo clean
